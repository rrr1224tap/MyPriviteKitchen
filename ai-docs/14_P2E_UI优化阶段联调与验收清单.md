# P2-E UI优化阶段联调与验收清单

文档日期：2026-06-22

阶段范围：P2-E UI 细节、空状态、错误状态、按钮状态和视觉一致性优化。

---

## 一、阶段结论

P2-E UI 细节、空状态、错误状态、按钮状态和视觉一致性优化已完成。

本阶段覆盖：

1. 用户端详情页错误状态收口。
2. 用户端菜单 fallback 提示优化。
3. 购物车下单错误提示和防重复优化。
4. 用户订单列表 / 订单详情状态展示优化。
5. 商家订单列表 / 订单详情空状态、错误状态和确认弹窗优化。
6. 全局视觉细节统一。

本阶段不新增业务能力，不修改数据库结构，不修改云函数接口，不改变订单状态流转规则。

---

## 二、完成范围

### 1. 用户菜单页

已完成：

1. fallback 菜单提示。
2. fallback 菜单重试入口。
3. 真实菜单加载成功时不显示 fallback 提示。
4. 菜单页视觉细节统一。
5. 菜名、分类、空状态和 fallback 文案的长文本保护。

### 2. 用户餐品详情页

已完成：

1. 缺少 `dish_id` 时不再展示可下单 fallback 餐品。
2. `getDishDetail` 失败时进入错误状态。
3. `getDishDetail` 返回空数据或无有效餐品 ID 时进入错误状态。
4. 错误状态不允许加入购物车。
5. 错误状态隐藏底部加入购物车栏。
6. 规格 / 加料选择逻辑保持不变。
7. 售罄 / 库存禁用逻辑保持不变。
8. 详情页视觉细节统一。

### 3. 用户购物车页

已完成：

1. 防重复提交。
2. 提交中按钮禁用并显示提交中状态。
3. 提交中阻止数量增减、移除、清空购物车和切换取餐方式。
4. `createOrder` 错误码友好提示。
5. 餐品失效类错误使用 modal 引导。
6. modal / toast 不再重叠。
7. 失败后不自动清空购物车。
8. 成功下单后仍清空购物车并进入订单查看流程。
9. 规格 / 加料提交结构保持不变。
10. 购物车视觉细节统一。

### 4. 用户订单列表 / 订单详情

已完成：

1. 订单状态文案统一。
2. `unknown` / 缺失状态兼容。
3. `cancelled` 状态详情进度兼容。
4. 错误文案转译，不直接展示技术错误字段。
5. 长菜名、长规格 / 加料、订单号、备注和金额区域布局保护。
6. 旧订单缺字段兼容。
7. 规格 / 加料展示保持不变。
8. 订单金额不在前端重新计算，仍使用订单快照金额。
9. 订单页视觉细节统一。

### 5. 商家订单列表 / 订单详情

已完成：

1. 状态操作二次确认。
2. 接单前确认。
3. 开始制作前确认。
4. 完成订单前确认。
5. 防重复操作。
6. 请求中按钮禁用并显示处理中状态。
7. 筛选空状态文案优化。
8. 错误文案转译，不直接展示技术错误字段。
9. 商家订单详情缺少 `order_id` 或接口失败时显示友好错误。
10. 规格 / 加料展示保持不变。
11. 商家订单页视觉细节统一。

### 6. 全局视觉

已完成：

1. 空状态 / 错误状态卡片视觉统一。
2. 主按钮、次按钮、提交按钮、状态按钮、禁用按钮细节统一。
3. 订单状态 badge、数量 `item-count`、售罄 badge 视觉统一。
4. 菜名、规格 / 加料、订单号、备注、订单摘要、金额说明等长文本保护。
5. 底部安全区结构保持。
6. 玻璃风 / 暖色商业餐饮风保持。
7. 没有改成普通白底后台风格。
8. 没有使用系统默认蓝色按钮。

---

## 七、风险与注意事项

1. P2-E 主要是页面体验优化，不改变业务数据结构。
2. P2-E 没有修改云函数，因此不需要重新部署云函数。
3. 如果出现样式错位，优先检查 WXSS 视觉提交。
4. 如果出现业务逻辑异常，需要确认是否误改 JS。
5. `.agents/` 和 `skills-lock.json` 为本地排除文件，不应提交。
6. 真机小屏幕仍建议再次预览检查。
7. 如果用户端或商家端出现技术错误字段，优先检查对应页面的错误文案转译逻辑。
8. 如果下单失败时同时出现 modal 和 toast，优先检查购物车页错误处理是否重复提示。

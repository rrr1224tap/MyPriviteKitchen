## 二、本阶段完成范围

### 2.1 数据库设计更新

dishes 集合正式采用以下库存 / 售罄字段：

```js
{
  stock_enabled: false,
  stock_count: 0,
  sold_out: false
}
```

字段说明：

1. `stock_enabled`：是否启用库存管理。
2. `stock_count`：剩余库存数量，必须是非负整数。
3. `sold_out`：手动售罄开关。

旧字段：

```text
stock
```

已标记为废弃字段，不再作为后续新功能主字段。

### 2.2 图片字段文档统一

餐品主图字段统一为：

```text
image_url
```

订单明细商品图片快照字段统一为：

```text
dish_image_url
```

### 2.3 manageDish

`manageDish` 已完成以下能力：

1. `create` 支持 `stock_enabled`、`stock_count`、`sold_out`。
2. `update` 支持 `stock_enabled`、`stock_count`、`sold_out`。
3. `list` 返回 `stock_enabled`、`stock_count`、`sold_out`。
4. 校验 `stock_enabled` 必须是 boolean。
5. 校验 `stock_count` 必须是非负整数。
6. 校验 `sold_out` 必须是 boolean。
7. 禁止使用 `status = sold_out`。
8. 保持餐品上下架状态仍使用 `on_sale` / `off_sale`。

### 2.4 getMenu

`getMenu` 已完成以下能力：

1. 返回每个餐品的 `stock_enabled`。
2. 返回每个餐品的 `stock_count`。
3. 返回每个餐品的 `sold_out`。
4. 售罄餐品仍然返回给前端。
5. 售罄餐品不从菜单中隐藏。
6. 仍然只返回 `status = on_sale` 的餐品。

### 2.5 getDishDetail

`getDishDetail` 已完成以下能力：

1. 返回餐品详情中的 `stock_enabled`。
2. 返回餐品详情中的 `stock_count`。
3. 返回餐品详情中的 `sold_out`。
4. 售罄餐品仍可查看详情。
5. 不新增 `status = sold_out`。

### 2.6 createOrder

`createOrder` 已完成以下能力：

1. 下单时重新查询 `dishes` 集合。
2. 校验餐品是否下架。
3. 校验餐品是否手动售罄。
4. 校验库存是否足够。
5. 支持重复 `dish_id` 合并数量后统一校验。
6. 下单成功后扣减库存。
7. 使用 CloudBase 条件更新 + `_.inc(-quantity)` 做单个餐品原子扣减。
8. `stock_enabled = false` 时不扣减库存。
9. 不信任前端传入的库存状态。
10. 不信任前端传入的价格和总金额。

### 2.7 商家餐品管理页

商家餐品管理页已完成以下能力：

1. 增加库存开关。
2. 增加库存数量输入。
3. 增加手动售罄开关。
4. 新增餐品时提交库存 / 售罄字段。
5. 编辑餐品时回显库存 / 售罄字段。
6. 列表展示库存状态。

列表状态包括：

```text
不限库存
剩余 X 份
已售罄
手动售罄
```

### 2.8 用户菜单页

用户菜单页已完成以下能力：

1. 显示“已售罄”标签。
2. 禁用售罄餐品加购。
3. 点击售罄餐品加购时提示“该餐品已售罄”。
4. 售罄餐品仍展示，不隐藏。
5. 写入购物车时不写入 `sold_out`。

### 2.9 用户餐品详情页

用户餐品详情页已完成以下能力：

1. 显示“已售罄”标签。
2. 禁用加入购物车按钮。
3. 售罄时禁止继续增加数量。
4. 点击售罄餐品加入购物车时提示“该餐品已售罄”。
5. 售罄餐品详情仍可正常查看。

## 五、手动联调结果

微信开发者工具手动联调 6 个场景全部通过。

### 5.1 商家设置库存

已验证：

1. 商家端可以设置 `stock_enabled = true`。
2. 商家端可以设置 `stock_count`。
3. 商家端可以设置 `sold_out = false`。
4. 保存后餐品管理页正确显示“剩余 X 份”。
5. 数据库 `dishes` 集合字段写入正确。

### 5.2 用户正常下单扣库存

已验证：

1. 用户端菜单页正常显示有库存餐品。
2. 用户可以加购。
3. 用户可以提交订单。
4. 订单创建成功。
5. `stock_count` 按购买数量正确扣减。

### 5.3 库存扣到 0 后自动显示已售罄

已验证：

1. 库存扣减到 0 后，数据库 `stock_count = 0`。
2. 用户菜单页仍展示该餐品。
3. 用户菜单页显示“已售罄”。
4. 用户详情页显示“已售罄”。
5. 用户不能继续加购该餐品。

### 5.4 库存不足时后端兜底拦截

已验证：

1. 当 `stock_enabled = true` 且库存小于下单数量时，`createOrder` 拒绝下单。
2. 返回错误码 `STOCK_NOT_ENOUGH`。
3. 数据库库存不会扣成负数。
4. 前端保留购物车内容，便于用户调整。

### 5.5 手动售罄

已验证：

1. 商家设置 `sold_out = true` 后，用户菜单页显示“已售罄”。
2. 用户详情页显示“已售罄”。
3. 用户不能加购。
4. 如果购物车中已有该餐品，提交订单时由 `createOrder` 拦截。
5. 返回错误码 `DISH_SOLD_OUT`。

### 5.6 不限库存

已验证：

1. 当 `stock_enabled = false` 且 `sold_out = false` 时，用户端可以正常加购和下单。
2. 下单成功后不扣减 `stock_count`。
3. 不限库存餐品不会被误判为售罄。

## 七、已知风险

1. `createOrder` 目前使用条件更新 + `_.inc(-quantity)` 做单个餐品原子扣减。
2. 订单创建、订单明细写入、多个餐品扣库存之间还不是完整事务。
3. 极端并发或中途失败时，后续可升级为数据库事务。
4. 购物车旧缓存中已有售罄餐品时，最终由 `createOrder` 拦截。
5. 前端售罄展示只作为体验优化，真实库存和售罄校验仍以后端 `createOrder` 为准。

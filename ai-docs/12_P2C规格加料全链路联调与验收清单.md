# P2-C 规格加料全链路联调与验收清单

文档日期：2026-06-22

阶段标签建议：P2-C 规格 / 加料闭环

---

## 一、阶段结论

P2-C 规格 / 加料核心链路已闭环。

本阶段已经覆盖以下能力：

1. 商家可以在餐品中配置规格 / 加料。
2. 用户菜单页对有规格 / 加料的餐品显示“选规格”入口。
3. 用户餐品详情页可以选择规格 / 加料。
4. 购物车按 `item_key` 区分同一餐品的不同规格 / 加料组合。
5. `createOrder` 云端校验规格 / 加料，并重新计算订单金额。
6. 用户端订单列表 / 订单详情展示规格 / 加料快照。
7. 商家端订单列表 / 订单详情展示规格 / 加料快照。

本阶段不接微信支付，不做退款、优惠券、打印机、多门店等能力。

## 三、关键数据结构

### 1. 餐品规格结构

```js
spec_groups: [
  {
    group_id,
    name,
    required,
    min_select,
    max_select,
    sort_order,
    options: [
      {
        option_id,
        name,
        price_delta_cent,
        enabled,
        sort_order
      }
    ]
  }
]
```

第一阶段规格组按单选处理，`max_select` 固定为 `1`。

### 2. 餐品加料结构

```js
addon_groups: [
  {
    group_id,
    name,
    required,
    min_select,
    max_select,
    sort_order,
    options: [
      {
        option_id,
        name,
        price_delta_cent,
        enabled,
        sort_order
      }
    ]
  }
]
```

加料组选项可以多选，选择数量必须满足 `min_select` / `max_select`。

### 3. 订单明细快照

```js
{
  dish_id,
  dish_name,
  dish_image_url,
  base_price_cent,
  spec_delta_cent,
  addon_delta_cent,
  unit_price_cent,
  selected_specs,
  selected_addons,
  quantity,
  subtotal_cent
}
```

订单展示使用订单快照，不反查当前餐品配置，避免商家后续改价影响历史订单。

### 4. 购物车 item_key 规则

```text
dish_id|spec:group_id=option_id|addon:group_id=option_id1,option_id2
```

规则：

1. 同一餐品 + 相同规格 / 加料，`item_key` 一致，购物车数量累加。
2. 同一餐品 + 不同规格 / 加料，`item_key` 不同，购物车分开显示。
3. 加料 `option_ids` 按固定顺序排序，保证 key 稳定。
4. 旧购物车数据没有 `item_key` 时，使用 `dish_id` 兼容。

## 四、金额安全原则

1. 前端不可信。
2. 前端不传可信价格给 `createOrder`。
3. `createOrder` 不信任前端传入的价格、加价、小计和总价。
4. `base_price_cent`、`spec_delta_cent`、`addon_delta_cent`、`unit_price_cent`、`subtotal_cent`、`total_amount_cent` 都由云函数读取数据库后重新计算。
5. 订单展示使用订单明细快照，不反查当前餐品配置，避免后续餐品修改影响历史订单。
6. 所有金额仍然使用整数分，不使用浮点金额保存。

计算公式：

```text
unit_price_cent = price_cent + spec_delta_cent + addon_delta_cent
subtotal_cent = unit_price_cent * quantity
total_amount_cent = 所有 subtotal_cent 相加
```

## 九、风险与注意事项

1. 修改规格 / 加料配置不会影响历史订单，因为历史订单使用订单快照。
2. 如果新增规格配置后用户端看不到，需要确认 `getMenu` / `getDishDetail` 已重新部署。
3. 如果订单里看不到规格 / 加料，需要确认 `createOrder`、`getOrderDetail`、`getUserOrders`、`getMerchantOrderDetail`、`getMerchantOrders` 都已重新部署。
4. 如果购物车出现重复 `wx:key`，需要确认购物车页使用 `item_key`。
5. 如果 `createOrder` 返回 `VALIDATION_ERROR`，需要检查 `selected_specs` / `selected_addons` 是否正确传递。
6. `.agents/` 和 `skills-lock.json` 属于本地 Agent / skill 缓存，已加入本地 `.git/info/exclude`，不应提交。
7. 规格 / 加料价格只用于用户端展示预估，最终仍以云函数重算结果为准。
8. 旧订单、旧购物车和旧餐品缺失规格 / 加料字段时，应按空数组兼容。

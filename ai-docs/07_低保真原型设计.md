---



\## 1. 文档目标



本文档用于规范微信点餐小程序的低保真原型结构，明确每个页面的模块顺序、信息层级、按钮位置、页面跳转关系和交互规则。



Codex 在开发页面前，必须先阅读本文档和：



```text

ai-docs/08\_UI视觉规范\_商业餐饮风.md

```



后续页面开发不允许只使用系统默认页面效果，也不允许自由发挥页面结构。



---



\## 2. 原型风格定位



本项目页面原型采用“商业餐饮风”。



核心感觉：



```text

明亮

干净

商业化

食物图突出

促销感明确

点餐效率高

类似成熟餐饮品牌小程序

```



参考方向：



```text

韩式拌饭 / 快餐 / 火锅 / 砂锅 / 套餐类点餐小程序

```



注意：



本文档只参考整体结构、商业感和视觉方向，不复制任何已有设计图中的素材、文字、图标、品牌、排版细节。

\## 4. 用户端整体结构



用户端采用“品牌首页 + 点餐菜单 + 订单闭环”的结构。



用户主路径：



```text

首页

↓

点餐菜单页

↓

餐品详情页 / 加入购物车

↓

购物车页

↓

提交订单页

↓

订单详情页

```



用户也可以从底部 TabBar 进入：



```text

首页

点餐

订单

我的 / 商家入口

```



MVP 阶段“我的”页面可以暂不单独开发，可先作为进入商家端的入口。

点击“订单”Tab -> 跳转用户订单列表页

点击“我的”Tab -> MVP 阶段可跳转商家首页或预留个人中心

```text

1\. 首页必须有大幅主视觉区域。

2\. 不要做成普通系统列表。

3\. 首页卡片要有商业餐饮品牌感。

4\. 不要使用默认蓝色按钮。

5\. 底部 TabBar 风格需与商业餐饮风一致。

```

\### 6.7 Codex 开发要求



```text

1\. 点餐页必须采用左侧分类 + 右侧商品列表结构。

3\. 购物车条必须固定在底部 TabBar 上方。

4\. 分类选中态必须明显。

5\. 商品按钮使用品牌红色，不使用系统蓝色。

```

支持分页加载

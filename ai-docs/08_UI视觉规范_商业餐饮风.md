## 1. 文档目标

本文档用于规范微信点餐小程序的 UI 视觉风格，确保 Codex 后续开发页面和组件时不会使用系统默认页面效果，也不会在不同页面中写出风格不统一的样式。

Codex 在开发任何页面和组件前，必须先阅读：

```text
ai-docs/07_低保真原型设计.md
ai-docs/08_UI视觉规范_商业餐饮风.md
```

## 3. 参考风格说明

当前视觉风格参考方向：

```text
韩式拌饭小程序
砂锅米线小程序
火锅套餐点餐小程序
连锁快餐品牌小程序
```

核心特点：

```text
1. 首页有强视觉食品大图。
2. 点餐页有顶部店铺信息。
3. 点餐页有会员条和促销 Banner。
4. 菜单主体为左侧分类 + 右侧商品卡片。
5. 商品卡片以大食物图为核心。
6. 加购按钮使用品牌红色。
7. 底部有明显购物车结算条。
8. 页面背景使用浅灰或米白，不是纯白。
```

注意：

只参考风格和结构，不复制参考图中的具体素材、文字、品牌、图标、插画或页面细节。

## 4. 色彩规范

### 4.1 主色

品牌主色使用红色，偏商业餐饮、促销、按钮导向。

```text
品牌红：#E63B4A
深红：#C92E3B
浅红背景：#FFF0F2
```

使用场景：

```text
主按钮
加购按钮
选中状态
订单状态重点提示
会员条
促销提示
```

---

### 4.2 辅助色

```text
背景浅灰：#F3F3F3
页面暖白：#FFF8EF
卡片白：#FFFFFF
文字黑：#171717
次级文字：#777777
弱文字：#AAAAAA
分割线：#EDEDED
绿色结算：#5A963B
橙色促销：#E7A043
黄色星级：#FFCB3E
危险提示：#D85A42
```

---

### 4.3 页面背景

默认页面背景：

```text
#F3F3F3
```

首页顶部或品牌区可以使用暖白渐变：

```text
#FFF8EF -> #F3F3F3
```

禁止：

```text
整页纯白无层次
整页高饱和红色
默认蓝色背景
```

## 5. 字体规范

### 5.1 字体族

微信小程序内优先使用系统字体：

```text
-apple-system
BlinkMacSystemFont
PingFang SC
Microsoft YaHei
Arial
sans-serif
```

不引入复杂外部字体。

---

### 5.2 字号层级

```text
页面主标题：34rpx - 40rpx，加粗
模块标题：28rpx - 32rpx，加粗
商品名称：28rpx - 30rpx，加粗
正文描述：22rpx - 26rpx
辅助文字：20rpx - 24rpx
价格：34rpx - 40rpx，加粗
按钮文字：26rpx - 30rpx，加粗
TabBar文字：20rpx - 22rpx
```

---

### 5.3 字重

```text
重要标题：800 - 900
商品名：800 - 900
按钮：800 - 900
正文：400 - 500
辅助说明：400
```

Codex 实现时，如 WXSS 不支持数值字重差异，可使用：

```css
font-weight: bold;
```

## 6. 圆角规范

商业餐饮风整体使用中等圆角，不要过度奶油化。

```text
页面大卡片：24rpx - 32rpx
商品卡片：20rpx - 26rpx
图片圆角：18rpx - 24rpx
小标签：999rpx
主按钮：999rpx
分类项：20rpx - 24rpx
首页入口卡片：24rpx - 32rpx
```

禁止：

```text
所有元素都直角
所有元素都过度大圆角导致幼态
```

---

## 7. 阴影规范

页面使用轻阴影，营造商业 App 卡片感。

推荐阴影：

```css
box-shadow: 0 8rpx 24rpx rgba(0, 0, 0, 0.06);
```

重点卡片：

```css
box-shadow: 0 12rpx 32rpx rgba(0, 0, 0, 0.08);
```

按钮阴影：

```css
box-shadow: 0 8rpx 20rpx rgba(230, 59, 74, 0.22);
```

禁止：

```text
过重阴影
高对比投影
所有元素都无阴影导致层次不足
```

## 9. 图片规范

### 9.1 食物图片风格

商品图片是本风格核心。

推荐：

```text
真实食物照片
明亮光线
食物色彩丰富
有锅具 / 餐具 / 桌面
近景拍摄
适当商业修图
突出热气、芝士、汤汁、酱料、米饭等食欲点
```

不推荐：

```text
普通图标占位
纯色块占位
过度 AI 感图片
过暗图片
杂乱背景图片
低清晰度图片
```

### 9.2 首页主视觉

首页主视觉建议使用：

```text
主推餐品大图
活动图
品牌招牌菜
汤锅 / 拌饭 / 套餐大图
```

主视觉需要具备强引导能力，类似广告 Banner 或活动海报。

## 16. WXML / WXSS 开发要求

Codex 开发页面时需要遵守：

```text
1. 样式尽量复用公共 class。
2. 不要每个页面重复写一套完全不同的按钮样式。
3. 不要使用默认 button 蓝色样式。
4. 不要使用浏览器默认 input 样式。
5. 金额、状态、按钮、卡片都应有统一 class。
6. 复杂样式优先沉淀到 app.wxss 或组件 wxss。
```

## 19. 禁止事项

Codex 禁止以下行为：

```text
1. 使用系统默认蓝色按钮。
2. 页面背景全白无层次。
3. 商品卡片没有商品图。
4. 商品列表只做成普通文字列表。
5. 每个页面视觉风格不一致。
6. 商家端使用过度花哨装饰影响效率。
7. 直接复制参考图中的品牌、文字、素材。
8. 随意引入第三方 UI 库。
9. 使用复杂外部字体。
10. 用 CSS 做过多无法在小程序 WXSS 中稳定实现的效果。
```

## 20. Codex 开发前检查清单

每次开发页面前，Codex 必须检查：

```text
1. 当前页面对应 07 原型中的哪个页面？
2. 当前页面有哪些模块？
3. 当前页面是否需要调用云函数？
4. 当前页面是否需要使用组件？
5. 当前页面是否使用商业餐饮风？
6. 是否使用品牌红作为主按钮？
7. 是否有食物图或图片占位？
8. 是否有空状态和 loading 状态？
9. 是否避免了系统默认样式？
```

## 22. 当前结论

本项目视觉风格确定为“商业餐饮风”。

核心不是做一个普通点餐系统，而是做一个更像成熟餐饮品牌小程序的点餐体验。

用户端要有：

```text
品牌感
促销感
食欲感
清晰点餐效率
```

商家端要有：

```text
清晰状态
高效操作
数据概览
订单处理效率
```

后续所有页面和组件开发，都必须以本文档作为视觉依据。

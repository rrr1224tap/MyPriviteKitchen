# Web 后台页面规划

## `/login` 登录页

页面目标：提供 Web 后台入口，完成后续真实登录前的入口承载。

页面用户：超级管理员。

页面内容区块：品牌区、登录卡片、管理口令输入框、登录按钮、本地静态预览提示。

主要操作：输入口令、点击登录、跳转后台首页。

空状态：不适用。

错误状态：后续登录失败时显示友好错误文案。

权限要求：未登录可访问；登录后跳转 `/`。

数据来源：v0.5-A 为静态数据；v0.5-B 接入登录会话。

后续设计重点：保持小厨食堂暖白、轻毛玻璃、品牌红按钮，不使用传统蓝色企业后台。

## `/` 总览页

页面目标：展示后台关键数据、风险提醒、最近订单和模块入口。

页面用户：超级管理员。

页面内容区块：欢迎区、版本提示、统计卡片、模块卡片、风险提醒、最近订单。

主要操作：刷新、进入模块、查看风险。

空状态：没有风险时显示“暂无风险提醒”；没有订单时显示“暂无最近订单”。

错误状态：数据加载失败时显示重试按钮。

权限要求：超级管理员。

数据来源：`getAdminOverview`。

后续设计重点：卡片密度适合电脑端扫描，风险提醒不抢主流程。

## `/merchants` 商户管理

页面目标：管理系统中的商户。

页面用户：超级管理员。

页面内容区块：筛选区、商户列表、商户表单、状态标签、操作区。

主要操作：创建商户、编辑商户、启用商户、禁用商户。

空状态：暂无商户，引导创建第一个商户。

错误状态：加载失败、保存失败、重复 `merchant_id`。

权限要求：超级管理员。

数据来源：`manageMerchant`。

后续设计重点：突出 `merchant_id` 的唯一性和不可轻易变更。

## `/merchants/:merchantId/staff` 成员与邀请

页面目标：管理某个商户的成员和邀请码。

页面用户：超级管理员。

页面内容区块：商户信息、成员列表、邀请码列表、生成邀请码表单。

主要操作：生成邀请码、复制邀请码、禁用邀请码、启用成员、禁用成员。

空状态：暂无成员、暂无邀请码。

错误状态：邀请码生成失败、成员状态修改失败。

权限要求：超级管理员。

数据来源：`manageMerchantStaff`。

后续设计重点：邀请码要清晰区分有效、已使用、已过期、已禁用。

## `/dishes` 餐品管理

页面目标：管理餐品基础信息、规格、加料、食材和做法参考。

页面用户：超级管理员、后续商户管理员。

页面内容区块：分类筛选、餐品列表、餐品表单、规格配置、加料配置、食材配置、做法参考。

主要操作：新增餐品、编辑餐品、上架、下架、排序、保存配置。

空状态：暂无餐品，引导新增。

错误状态：加载失败、保存失败、字段校验失败。

权限要求：商户级权限或超级管理员。

数据来源：`manageDish`、`getMenu`、`getDishDetail`。

后续设计重点：表单分组必须清晰，不能让规格、加料、食材和做法参考互相挤压。

## `/categories` 分类管理

页面目标：管理餐品分类。

页面用户：超级管理员、后续商户管理员。

页面内容区块：分类列表、分类表单、排序操作、启用状态。

主要操作：新增分类、编辑分类、启用、停用、上移、下移。

空状态：暂无分类，引导新增。

错误状态：加载失败、保存失败、排序失败。

权限要求：商户级权限或超级管理员。

数据来源：`manageCategory`。

后续设计重点：排序按钮弱化，启用状态清晰。

## `/orders` 订单管理

页面目标：查看订单列表并进入详情。

页面用户：超级管理员、后续商户管理员。

页面内容区块：状态筛选、日期筛选、订单列表、订单摘要、操作区。

主要操作：筛选订单、查看详情、刷新。

空状态：当前筛选下暂无订单。

错误状态：加载失败，提供重试。

权限要求：商户级权限或超级管理员。

数据来源：`getMerchantOrders`。

后续设计重点：列表要适合快速识别待处理订单，但不新增订单状态。

## `/orders/:orderId` 订单详情

页面目标：查看订单完整信息、商品明细、规格加料快照和做法参考。

页面用户：超级管理员、后续商户管理员。

页面内容区块：订单状态、取餐信息、商品明细、金额明细、做法参考、备注。

主要操作：复制做法参考、返回订单列表。

空状态：订单不存在或已被删除。

错误状态：无权限、加载失败。

权限要求：商户级权限或超级管理员。

数据来源：`getMerchantOrderDetail`。

后续设计重点：第一版可先规划为抽屉或弹窗，避免页面跳转过重。

## `/prep-summary` 今日备料

页面目标：按今日订单汇总食材。

页面用户：超级管理员、后续商户管理员。

页面内容区块：日期说明、汇总列表、分类分组、复制采购清单按钮。

主要操作：刷新、复制采购清单。

空状态：暂无今日订单或暂无可汇总食材。

错误状态：加载失败、无权限。

权限要求：商户级权限或超级管理员。

数据来源：`getPrepSummary`。

后续设计重点：数量、单位、分类要清楚，不做复杂库存系统。

## `/data-health` 数据检查

页面目标：检查关键数据完整性并执行安全轻量修复。

页面用户：超级管理员。

页面内容区块：检查结果、问题列表、可修复项、修复确认弹窗。

主要操作：执行检查、确认修复缺失 `merchant_id`。

空状态：暂无问题。

错误状态：检查失败、修复失败。

权限要求：超级管理员。

数据来源：`checkAdminDataHealth`。

后续设计重点：修复动作必须有明确边界和二次确认。

## `/settings` 系统设置

页面目标：展示系统配置和上线前检查项。

页面用户：超级管理员。

页面内容区块：环境信息、默认商户说明、云函数部署清单、版本信息。

主要操作：第一版只读，后续再规划编辑。

空状态：不适用。

错误状态：配置读取失败。

权限要求：超级管理员。

数据来源：后续确定。

后续设计重点：不显示敏感密钥，不开放危险配置。

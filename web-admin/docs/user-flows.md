# Web 后台用户流程

## 1. 超级管理员登录流程

1. 打开 Web 后台 `/login`。
2. 输入管理口令或后续登录凭证。
3. 点击登录。
4. 前端请求登录或会话校验能力。
5. 校验通过后进入 `/`。
6. 校验失败时停留在登录页并显示友好提示。

## 2. 查看后台总览流程

1. 超级管理员进入 `/`。
2. 页面加载 `getAdminOverview`。
3. 展示统计卡片、风险提醒和最近订单。
4. 用户点击手动刷新时重新拉取数据。
5. 加载失败时显示重试按钮。

## 3. 创建商户流程

1. 进入 `/merchants`。
2. 点击创建商户。
3. 填写 `merchant_id`、商户名称、联系人、备注等基础信息。
4. 提交到 `manageMerchant` 的 `create` action。
5. 创建成功后刷新列表。
6. 如果 `merchant_id` 重复，展示友好错误。

## 4. 编辑 / 禁用商户流程

1. 进入 `/merchants`。
2. 选择一个商户。
3. 编辑基础信息并保存，调用 `manageMerchant update`。
4. 点击禁用或启用时弹出二次确认。
5. 确认后调用 `disable` 或 `enable`。
6. 成功后刷新商户状态。

## 5. 生成邀请码流程

1. 进入 `/merchants/:merchantId/staff`。
2. 选择生成邀请码。
3. 填写有效期、备注或角色范围。
4. 调用 `manageMerchantStaff` 生成邀请码。
5. 页面展示邀请码并支持复制。

## 6. 普通用户兑换邀请码流程

1. 普通用户获得邀请码。
2. 在小程序端或后续 Web 入口输入邀请码。
3. 调用 `redeemMerchantInvite`。
4. 云函数校验邀请码状态、有效期和使用次数。
5. 成功后写入 `merchant_staff`。
6. 再次进入商家工作台时拥有商家身份。

## 7. 管理成员流程

1. 进入成员与邀请页。
2. 查看当前商户成员列表。
3. 对成员执行启用或禁用。
4. 操作前二次确认。
5. 调用 `manageMerchantStaff` 更新成员状态。
6. 成功后刷新成员列表。

## 8. 新增 / 编辑餐品流程

1. 进入 `/dishes`。
2. 点击新增餐品或编辑餐品。
3. 填写基础信息、分类、价格、库存。
4. 配置规格、加料、食材和做法参考。
5. 保存时调用 `manageDish`。
6. 成功后刷新餐品列表。
7. 老餐品缺少 `ingredients` 或 `tutorials` 时按空数组处理。

## 9. 配置食材与做法参考流程

1. 在餐品表单中打开食材配置区域。
2. 添加食材名称、数量、单位、分类、启用状态。
3. 打开做法参考区域。
4. 添加标题、平台、链接或口令、备注、启用状态。
5. 保存餐品。
6. 用户端不展示食材和做法参考。
7. 商家订单详情展示做法参考，今日备料读取启用食材。

## 10. 查看订单流程

1. 进入 `/orders`。
2. 选择状态或日期筛选。
3. 查看订单摘要、金额和取餐方式。
4. 点击订单进入详情或打开抽屉。
5. 查看商品明细、规格加料快照、备注和做法参考。

## 11. 查看今日备料流程

1. 进入 `/prep-summary`。
2. 页面调用 `getPrepSummary`。
3. 系统按今日非取消订单汇总启用食材。
4. 同名同单位食材合并。
5. 同名不同单位食材分开显示。
6. 用户点击复制采购清单。

## 12. 数据检查与轻量修复流程

1. 进入 `/data-health`。
2. 点击执行检查。
3. 调用 `checkAdminDataHealth check`。
4. 页面展示问题和可修复项。
5. 用户点击修复前出现二次确认。
6. 确认后只执行白名单修复：补齐缺失或空的 `dishes.merchant_id`、`categories.merchant_id`。
7. 修复完成后重新检查。

## 13. 退出登录流程

1. 点击顶部栏退出按钮。
2. 清除本地临时会话。
3. 跳转 `/login`。
4. 不保留真实 token。

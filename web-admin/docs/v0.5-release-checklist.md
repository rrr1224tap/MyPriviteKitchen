# Private-Kitchen-v0.5 发布前总验收与上线清单

适用阶段：`Private-Kitchen-v0.5-G：正式上线前最后补齐与总验收`

本文用于记录 v0.5 发布前的总体验收范围、上线配置、云端部署、禁止能力、测试结果和手动验收步骤。本阶段只做发布前检查与文档沉淀，不新增业务功能，不执行新的真实写入。

## 1. 当前版本目标

v0.5 目标是让小厨食堂 Web 管理后台形成可预览、可部署、可验收的基础管理闭环：

1. Web 管理员可以登录后台并保持 session。
2. Web 后台可以读取总览、商户、成员、邀请码、分类、餐品、订单和今日备料真实数据。
3. Web 后台可以完成已开放的商户、成员、邀请码、分类、餐品和订单状态管理能力。
4. 小程序原有点餐链路和商家端链路保持稳定。
5. 发布前明确环境变量、HTTP 网关、云函数部署和禁止提交文件。

## 6. 鉴权与安全边界

1. Web 请求统一通过 `callAdminFunction` 或现有统一请求 helper。
2. Web 请求携带 `admin_token`。
3. 云函数内部通过 `WEB_ADMIN_TOKEN_SECRET` 校验 token。
4. token 必须通过签名校验。
5. token 必须通过过期时间校验。
6. token role 必须为 `super_admin`。
7. CloudBase HTTP 网关身份认证需要关闭，真实权限由云函数内部 token 校验。
8. 小程序 / 商家端原有 `OPENID + merchant_staff active` 权限逻辑保留。
9. 不依赖前端路由守卫作为唯一权限。
10. 不在前端源码写入 `WEB_ADMIN_TOKEN_SECRET`。
11. `.env.local` 不提交 Git。

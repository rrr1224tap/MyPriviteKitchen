# Web 后台设计简报

## 1. 整体视觉方向

小厨食堂 Web 后台应延续小程序 v0.2 以来确认的暖白、奶油白、轻毛玻璃风格。它是管理工具，但不应该变成传统企业蓝后台，也不应该变成深色科技后台。

关键词：奶油白背景、轻毛玻璃卡片、柔和阴影、品牌红主按钮、橙色辅助提示、绿色成功状态、圆角卡片、温暖、干净、现代、有私厨工具的亲和感。

## 2. 页面截图建议

建议后续给 image2 逐页输出高保真设计：

1. 登录页。
2. 总览页。
3. 商户管理页。
4. 成员与邀请页。
5. 餐品管理页。
6. 订单管理页。
7. 今日备料页。

9. 系统设置页。

每张截图都应保留页面真实信息结构，不做纯营销海报。

## 3. image2 设计提示词

请参考该 Web 后台页面截图，将其升级为“小厨食堂 Web 管理后台”的高保真视觉设计。请保留页面信息架构和内容层级，整体风格为奶油白背景、轻毛玻璃卡片、柔和阴影、品牌红主按钮、橙色辅助提示、绿色成功状态。视觉要温暖、干净、现代，有私厨工具的亲和感，不要传统企业蓝后台，不要深色后台，不要过度科技感。

## 5. 卡片规范

1. 卡片背景使用半透明暖白或浅米色渐变。
2. 圆角建议 18px 到 28px。
3. 阴影柔和，不使用硬阴影。
4. 边界使用浅色高光。
5. 重要卡片可使用更明显的内高光。

参考样式：

```css
background: linear-gradient(145deg, rgba(255,255,255,0.86), rgba(255,248,238,0.62));
border: 1px solid rgba(255,255,255,0.88);
box-shadow: 0 20px 56px rgba(126, 90, 45, 0.14), inset 0 1px 0 rgba(255,255,255,0.95);
backdrop-filter: blur(22px) saturate(150%);
```

## 6. 表格规范

1. 表格不使用密集灰线。
2. 行高充足，适合运营者长时间查看。
3. 表头使用浅暖色背景。
4. 操作按钮靠右，主次分明。
5. 状态用标签表达，不只靠文字。
6. 空状态要有明确说明和下一步动作。

## 7. 表单规范

1. 表单分组清晰：基础信息、业务配置、状态和备注。
2. 输入框使用浅白背景和柔和边框。
3. 必填字段明确标注。
4. 错误提示靠近字段显示。
5. 保存按钮使用品牌红。
6. 危险操作必须远离主保存按钮。

## 8. 状态标签规范

1. 启用、成功、已完成：绿色。
2. 待处理、制作中、提醒：橙色。
3. 重要主状态：品牌红或浅红。
4. 禁用、取消、错误：灰红或浅灰。
5. 标签使用圆角胶囊，不使用系统默认蓝色。

## 9. 按钮规范

1. 主按钮使用品牌红渐变。
2. 次按钮使用浅色毛玻璃。
3. 危险按钮使用柔和灰红，不用刺眼红。
4. 禁用态降低透明度。
5. 不使用系统默认蓝色按钮。

## 10. 不希望出现的风格

1. 传统企业蓝后台。
2. 深色科技后台。
3. 纯黑白极简后台。
4. 大面积高饱和红色。
5. 复杂装饰背景。
6. 营销海报式首页。
7. 默认模板页面样式。
8. 与小程序视觉完全割裂的 UI。

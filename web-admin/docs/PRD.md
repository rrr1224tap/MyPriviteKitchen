# 小厨食堂 Web 管理后台 PRD

## 1. 项目背景

小厨食堂已经完成微信小程序端的用户点餐、商家接单、餐品维护、做法参考、食材配置、今日备料、系统后台入口、商户管理、成员邀请码、后台概览和数据检查能力。小程序内后台适合移动端临时处理，但在批量录入餐品、查看多维数据、管理商户和做上线前检查时，电脑端 Web 后台会更高效。

v0.5 阶段的目标是在不破坏小程序现有业务链路的前提下，为超级管理员和私厨运营者提供一个电脑端管理工具，为正式上线版提供后台支撑。

## 2. 产品定位

小厨食堂 Web 管理后台是面向超级管理员和私厨运营者的电脑端管理工具，用于更高效地管理商户、成员、邀请码、餐品、订单、备料和数据健康，为小程序正式上线提供后台支撑。

## 3. 目标用户

1. 超级管理员：负责系统级商户、成员、数据健康和风险检查。
2. 私厨运营者：负责餐品、分类、订单、做法参考、食材配置和今日备料。
3. 后续可能加入的协作成员：负责有限范围的餐品维护或订单处理。

## 4. 使用场景

1. 上线前，超级管理员检查商户、成员、邀请码和关键数据是否完整。
2. 日常运营中，私厨运营者用电脑批量维护餐品、规格、加料、食材和做法参考。
3. 出餐前，运营者查看今日备料汇总并复制采购清单。
4. 接单高峰期，运营者在电脑端查看订单列表和订单详情。
5. 数据异常时，超级管理员执行只读检查，并在明确确认后做轻量修复。

## 5. v0.5 版本目标

在不破坏小程序现有业务链路的前提下，基于现有云函数和数据库能力，搭建电脑端 Web 管理后台，实现登录、总览、商户管理、成员邀请、餐品管理、订单查看、今日备料、数据检查等核心管理能力。

## 6. 功能范围

1. Web 后台登录与管理会话。
2. 后台总览，包含关键指标、风险提醒和最近订单。
3. 商户列表、商户创建、商户编辑、启用和禁用。
4. 商户成员与邀请码管理。
5. 餐品与分类管理。
6. 订单列表和订单详情查看。
7. 今日备料汇总和采购清单复制。
8. 数据健康检查与轻量修复。
9. 系统设置的基础规划。

## 7. 不做事项

1. 不做微信支付。
2. 不做退款。
3. 不做优惠券。
4. 不做会员。
5. 不做复杂多门店经营系统。
6. 不做打印机。
7. 不做配送系统。
8. 不做复杂财务报表。
9. 不做任意数据库 CRUD。
10. 不做危险批量删除。
11. 不重写小程序端业务链路。

## 8. 权限范围

1. Web 后台第一阶段默认只开放给超级管理员。
2. 超级管理员身份沿用 `SUPER_ADMIN_OPENIDS` 环境变量校验。
3. 普通用户不能访问 Web 后台。
4. 普通商家默认不能访问系统级页面。
5. 后续可扩展商户级后台权限，但必须继续通过云函数校验。

## 9. 数据范围

Web 后台优先复用现有集合：

1. `merchants`
2. `merchant_staff`
3. `merchant_invites`
4. `categories`
5. `dishes`
6. `orders`
7. `order_items`
8. `users`

v0.5-A1 只做文档规划，不新增集合，不修改字段。

## 10. 核心模块

1. 登录与会话：完成 Web 端身份进入方式。
2. 总览 Dashboard：查看系统关键数据和风险提醒。
3. 商户管理：管理商户基础信息和启停状态。
4. 成员与邀请：生成邀请码、查看成员、启用或禁用成员。
5. 餐品管理：维护餐品、规格、加料、食材和做法参考。
6. 分类管理：维护餐品分类和排序。
7. 订单管理：查看订单列表、筛选状态、查看详情。
8. 今日备料：按今日订单汇总食材。
9. 数据检查：检查关键数据完整性并支持轻量修复。
10. 系统设置：沉淀默认商户、环境、上线检查等配置入口。

## 11. 验收标准

1. Web 后台页面信息架构清晰。
2. 每个页面的目标、数据来源、权限要求和错误状态明确。
3. 可复用云函数和需要改造的接口明确。
4. 视觉设计方向能直接交给 image2 做页面设计。
5. 后续开发任务能按版本拆分执行。
6. 不修改小程序业务代码。
7. 不修改云函数业务代码。
8. 不新增 npm 依赖。

## 12. 风险点

1. Web 端登录方式尚未最终确定，需要避免把 openid 或权限写死在前端。
2. 小程序云函数多数依赖微信云上下文，Web 端调用方式需要单独设计。
3. 系统级后台权限必须严控，不能让普通商家访问超级管理员能力。
4. 餐品、订单、金额、规格和加料逻辑不能在 Web 端重新实现一套不一致规则。
5. 数据修复功能只能做白名单内轻量修复，避免危险批量写入。

## 13. 后续版本规划

2. v0.5-C：Web 后台总览接入。
3. v0.5-D：Web 商户与成员管理接入。
4. v0.5-E：Web 餐品与分类管理接入。
5. v0.5-F：Web 订单与备料管理接入。
6. v0.5-G：Web 数据检查接入。
7. v0.5-H：正式上线前检查与部署。

# Web 后台接口映射

## 阶段说明

Web 端登录与身份校验方式尚未最终接入，本阶段只做接口映射，不改云函数。后续接入时必须确认 Web 端如何安全获得用户身份，不能在前端写死 openid。

## `getAdminProfile`

Web 页面用途：登录后校验是否为超级管理员，决定是否允许进入系统后台。

请求 action：无或默认 profile。

核心参数：无，身份来自云函数上下文或后续 Web 会话。

返回数据：`is_super_admin`、`openid`、权限信息。

权限要求：`SUPER_ADMIN_OPENIDS` 环境变量包含当前 openid。

是否需要改造：需要在 v0.5-B 明确 Web 调用身份来源。

风险点：不能把超级管理员 openid 写在前端。

## `manageMerchant`

Web 页面用途：商户列表、创建、编辑、启用、禁用。

请求 action：`list`、`get`、`create`、`update`、`enable`、`disable`。

核心参数：`merchant_id`、商户名称、状态、联系人、备注等。

返回数据：商户列表、商户详情、操作结果。

权限要求：超级管理员。

是否需要改造：优先复用，Web 端补齐表单和错误展示。

风险点：`merchant_id` 不能重复，禁用商户不能误删数据。

## `manageMerchantStaff`

Web 页面用途：成员列表、邀请码列表、生成邀请码、禁用邀请码、成员启用和禁用。

请求 action：以当前云函数实现为准，至少覆盖成员和邀请管理。

核心参数：`merchant_id`、成员 id、邀请码 id、有效期、状态。

返回数据：成员列表、邀请码列表、操作结果。

权限要求：超级管理员。

是否需要改造：可能需要补充 Web 友好的分页和筛选。

风险点：不能把普通成员提升为超级管理员。

## `redeemMerchantInvite`

Web 页面用途：规划普通用户兑换邀请码流程；第一版主要用于小程序端。

请求 action：兑换邀请码。

核心参数：邀请码。

返回数据：兑换结果、商户成员身份。

权限要求：普通登录用户。

是否需要改造：Web 端暂不接入，后续视登录方式决定。

风险点：邀请码过期、已禁用、重复兑换要有明确提示。

## `getAdminOverview`

Web 页面用途：后台总览。

请求 action：无或默认 overview。

核心参数：可选时间范围。

返回数据：统计卡片、风险提醒、最近订单。

权限要求：超级管理员。

是否需要改造：优先复用。

风险点：必须只读数据库，不做修复。

## `checkAdminDataHealth`

Web 页面用途：数据检查与轻量修复。

请求 action：`check`、`fixDishMerchantId`、`fixCategoryMerchantId`。

核心参数：默认商户 id、修复 action。

返回数据：检查结果、问题列表、修复结果。

权限要求：超级管理员。

是否需要改造：优先复用。

风险点：修复动作不能覆盖已有 `merchant_id`，不能修改订单数据。

## `manageDish`

Web 页面用途：餐品创建和编辑。

请求 action：以现有实现为准，覆盖列表、创建、更新、上下架、排序。

核心参数：餐品基础信息、分类、价格、库存、规格、加料、`ingredients`、`tutorials`。

返回数据：餐品列表、保存结果。

权限要求：商户员工或超级管理员路径。

是否需要改造：可能需要明确超级管理员跨商户管理时的权限模型。

风险点：不能改变价格、规格、加料和库存逻辑。

## `manageCategory`

Web 页面用途：分类创建、编辑、启用、停用、排序。

请求 action：以现有实现为准。

核心参数：`merchant_id`、分类名称、排序、状态。

返回数据：分类列表、保存结果。

权限要求：商户员工或超级管理员路径。

是否需要改造：可能需要补充 Web 端列表读取的分页或排序。

风险点：不能影响历史订单。

## `getMenu`

Web 页面用途：读取菜单和餐品列表，可用于餐品管理的只读列表或预览。

请求 action：无。

核心参数：`merchant_id`、分类筛选。

返回数据：分类和已上架餐品。

权限要求：用户端可读；后台使用时仍应按商户隔离。

是否需要改造：后台管理不应只依赖用户端菜单，因为用户端只展示上架餐品。

风险点：下架餐品在后台也需要可见，必要时用 `manageDish` 列表替代。

## `getDishDetail`

Web 页面用途：读取餐品详情，辅助编辑表单初始化。

请求 action：无。

核心参数：`dish_id`。

返回数据：餐品详情、规格、加料。

权限要求：用户端可读已上架餐品；后台编辑需要更完整权限。

是否需要改造：后台编辑不建议完全依赖用户端详情接口。

风险点：用户端接口可能不返回后台需要的隐藏字段。

## `getMerchantOrders`

Web 页面用途：订单列表。

请求 action：无或现有列表查询。

核心参数：状态、分页、日期范围、`merchant_id`。

返回数据：订单列表、分页信息。

权限要求：商户员工。

是否需要改造：可能需要 Web 端日期筛选和分页。

风险点：不能返回其他商户订单。

## `getMerchantOrderDetail`

Web 页面用途：订单详情和做法参考展示。

请求 action：无。

核心参数：`order_id`。

返回数据：订单详情、商品明细、规格加料快照、做法参考。

权限要求：商户员工。

是否需要改造：优先复用。

风险点：普通用户不能看到做法参考。

## `getPrepSummary`

Web 页面用途：今日备料。

请求 action：无。

核心参数：日期或默认今日、`merchant_id`。

返回数据：按食材名称和单位合并后的备料清单。

权限要求：商户员工。

是否需要改造：可能需要支持 Web 端日期选择，但 v0.5 第一版先看今日。

风险点：取消订单必须排除。

## `login`

Web 页面用途：暂不直接复用。

请求 action：小程序登录。

核心参数：无。

返回数据：微信 openid 和用户信息。

权限要求：微信小程序上下文。

是否需要改造：Web 端不能直接照搬小程序登录。

风险点：Web 端身份方案需要单独设计。

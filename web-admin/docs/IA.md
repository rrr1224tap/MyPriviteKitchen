# Web 后台信息架构

## 总体原则

Web 后台采用“系统级管理 + 商户运营工具”的结构。一级导航保持稳定，避免把所有能力堆在总览页。每个模块只做自己的核心任务，危险操作必须有二次确认。

## 一级导航

1. 总览 Dashboard
2. 商户管理 Merchants
3. 成员与邀请 Staff & Invites
4. 餐品管理 Dishes
5. 分类管理 Categories
6. 订单管理 Orders
7. 今日备料 Prep Summary
8. 数据检查 Data Health
9. 系统设置 Settings

## 1. 总览 Dashboard

页面目标：让超级管理员快速看到系统当前健康度、关键规模、风险提醒和最近订单。

主要用户：超级管理员。

主要数据：商户数量、成员数量、餐品数量、今日订单、风险提醒、最近订单。

主要操作：刷新数据、进入商户管理、进入数据检查、查看最近订单。

关联云函数：`getAdminOverview`。

风险点：总览必须只读，不能在总览里直接修复或批量改数据。

开发优先级：P0。

## 2. 商户管理 Merchants

页面目标：创建、查看、编辑、启用和禁用商户。

主要用户：超级管理员。

主要数据：`merchants`。

主要操作：列表、详情、创建、编辑、启用、禁用。

关联云函数：`manageMerchant`。

风险点：`merchant_id` 是业务隔离关键字段，不能随意修改或重复。

开发优先级：P0。

## 3. 成员与邀请 Staff & Invites

页面目标：管理商户成员和邀请码，支持成员加入商户。

主要用户：超级管理员，后续可扩展为商户管理员。

主要数据：`merchant_staff`、`merchant_invites`。

主要操作：成员列表、启用成员、禁用成员、生成邀请码、禁用邀请码。

关联云函数：`manageMerchantStaff`、`redeemMerchantInvite`。

风险点：邀请码不能泄露系统级权限；成员权限变化必须由云函数校验。

开发优先级：P0。

## 4. 餐品管理 Dishes

页面目标：在电脑端高效维护餐品基础信息、规格、加料、食材配置和做法参考。

主要用户：私厨运营者、超级管理员。

主要数据：`dishes`、`categories`。

主要操作：列表、筛选、新增、编辑、上架、下架、排序、保存食材和做法参考。

关联云函数：`manageDish`、`getMenu`、`getDishDetail`。

风险点：不能在前端自行计算价格规则；规格、加料、金额逻辑必须沿用后端。

开发优先级：P1。

## 5. 分类管理 Categories

页面目标：维护餐品分类、排序和启用状态。

主要用户：私厨运营者、超级管理员。

主要数据：`categories`。

主要操作：新增、编辑、启用、停用、上移、下移。

关联云函数：`manageCategory`。

风险点：分类停用后不能误影响历史订单。

开发优先级：P1。

## 6. 订单管理 Orders

页面目标：查看订单列表、订单状态和订单详情，辅助运营者处理出餐。

主要用户：私厨运营者、超级管理员。

主要数据：`orders`、`order_items`。

主要操作：状态筛选、查看详情、查看商品明细、查看做法参考。

关联云函数：`getMerchantOrders`、`getMerchantOrderDetail`。

风险点：v0.5 第一阶段以查看为主，不在 Web 端扩展新的订单状态流转。

开发优先级：P1。

## 7. 今日备料 Prep Summary

页面目标：按今日订单和餐品食材配置汇总采购清单。

主要用户：私厨运营者、超级管理员。

主要数据：今日非取消订单、`dishes.ingredients`。

主要操作：刷新、复制采购清单、查看分类汇总。

关联云函数：`getPrepSummary`。

风险点：取消订单不能计入；同名不同单位不能合并。

开发优先级：P1。

## 8. 数据检查 Data Health

页面目标：检查关键数据完整性，并允许超级管理员执行安全白名单内的轻量修复。

主要用户：超级管理员。

主要数据：`merchants`、`merchant_staff`、`categories`、`dishes`、`orders`。

主要操作：检查、查看问题、确认修复 `dishes.merchant_id`、确认修复 `categories.merchant_id`。

关联云函数：`checkAdminDataHealth`。

风险点：修复动作不能覆盖已有 `merchant_id`，不能修改订单金额和状态。

开发优先级：P0。

## 9. 系统设置 Settings

页面目标：集中展示环境、默认商户、上线检查说明和后续系统配置。

主要用户：超级管理员。

主要数据：环境变量说明、默认商户配置、版本信息。

主要操作：第一版只读展示，后续再开放配置。

关联云函数：待定，优先不新增。

风险点：不要把敏感密钥写入前端，不提供危险配置入口。

开发优先级：P2。

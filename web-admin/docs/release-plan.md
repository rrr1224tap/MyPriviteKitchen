# Web 后台 v0.5 版本计划

## v0.5-B：Web 后台登录与管理会话

目标：让 Web 后台从静态登录过渡到可控的管理会话。

范围：登录方式设计、会话保存、退出登录、登录态守卫、无权限提示。

不做事项：不做复杂账号体系，不做微信支付，不接会员，不开放普通用户后台。

验收标准：未登录访问后台会跳转登录；登录后可进入总览；退出后无法访问后台页面。

依赖关系：需要确认 Web 端如何安全获取或映射超级管理员身份。

## v0.5-C：Web 后台总览接入

目标：把静态总览接入真实 `getAdminOverview`。

范围：统计卡片、风险提醒、最近订单、手动刷新、加载和错误状态。

不做事项：不做数据修复，不做复杂图表，不引入大型图表库。

验收标准：超级管理员能看到真实总览数据；普通用户不可见；失败时可重试。

依赖关系：依赖 v0.5-B 的会话和权限。

## v0.5-D：Web 商户与成员管理接入

目标：接入商户、成员和邀请码管理。

范围：商户列表、创建、编辑、启用、禁用；成员列表；邀请码生成、复制和禁用。

不做事项：不做多级角色体系，不做批量删除，不做财务结算。

验收标准：超级管理员能管理商户和邀请码；重复商户 id 有友好提示；成员状态可控。

依赖关系：依赖 `manageMerchant`、`manageMerchantStaff`、`redeemMerchantInvite`。

## v0.5-E：Web 餐品与分类管理接入

目标：在电脑端维护餐品、分类、规格、加料、食材和做法参考。

范围：餐品列表、餐品表单、分类管理、状态管理、排序。

不做事项：不改价格计算逻辑，不改规格加料结构，不新增库存系统。

验收标准：Web 保存后的餐品在小程序端正常展示和下单；食材与做法参考正常保存。

依赖关系：依赖 `manageDish`、`manageCategory`，并保持小程序端回归。

## v0.5-F：Web 订单与备料管理接入

目标：让运营者能在电脑端查看订单和今日备料。

范围：订单列表、订单详情、做法参考复制、今日备料、采购清单复制。

不做事项：第一版不新增 Web 端订单状态流转，不做打印机，不做配送。

验收标准：订单数据隔离正确；取消订单不计入备料；备料合并规则正确。

依赖关系：依赖 `getMerchantOrders`、`getMerchantOrderDetail`、`getPrepSummary`。

## v0.5-G：Web 数据检查接入

目标：接入数据健康检查和安全轻量修复。

范围：检查结果、问题列表、可修复项、二次确认、修复后重新检查。

不做事项：不做任意 CRUD，不做订单数据修复，不做批量删除。

验收标准：`check` 只读；修复只补齐缺失或空的 `merchant_id`；不覆盖已有值。

依赖关系：依赖 `checkAdminDataHealth` 和超级管理员权限。

## v0.5-H：正式上线前检查与部署

目标：完成 Web 后台上线前回归和部署准备。

范围：构建检查、权限回归、小程序链路回归、云函数部署清单、包体积检查、上线说明。

不做事项：不临时新增大功能，不引入未验证依赖。

验收标准：Web 构建通过；小程序核心链路通过；后台权限通过；体验版和 Web 预览可用。

依赖关系：依赖 v0.5-B 到 v0.5-G 完成。
